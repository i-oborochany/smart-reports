# CLAUDE.md — «Умные отчёты»

Приложение для МоегоСклада: формирует отчёты по расписанию и шлёт в Telegram / e-mail, показывает в приложении.
Стек: NestJS + React + PostgreSQL + Redis (BullMQ), Docker на FirstVDS.

- Спецификация: `docs/superpowers/specs/2026-06-14-smart-reports-design.md`
- План реализации: `docs/superpowers/plans/2026-06-14-smart-reports-implementation-plan.md`

## Две поверхности продукта

1. **Маркетинговый сайт** — есть раздел «Инструкция».
2. **Встроенное приложение/виджет** (iframe в МС) — вкладки «Вопрос-Ответ» и «Обновления».

## ПРАВИЛО: справка = единый источник (обязательно)

Инструкция, FAQ и «История обновлений» обязаны быть **1:1 на сайте и в приложении**. Источник **один**. Две копии разойдутся — не держим копий.

**Источник:** пакет `packages/help-content` (`src/index.tsx`), экспортирует нейтральными тегами (`<p>/<ul>/<li>/<b>`, без классов приложения):
- `helpSections` — разделы инструкции
- `helpFaq` — вопрос-ответ
- `helpChangelog` — история обновлений (новое сверху, с датой)
- `supportEmail`

**Подключение:**
- Сайт → страница «Инструкция» рендерит `helpSections` + `helpFaq` + `helpChangelog`.
- Приложение → «Вопрос-Ответ» = `helpFaq`, «Обновления» = `helpChangelog`. Локальные файлы **реэкспортируют** из пакета, не копируют.
- Docker каждого приложения копирует пакет (package.json до install, всю папку до build).

**Когда и куда писать:**
| Раздел | Когда | Что |
|---|---|---|
| Инструкция | появилась/изменилась заметная возможность | раздел простым языком: что/зачем/как |
| FAQ | повторяющийся вопрос/нюанс/типовая ошибка | короткий вопрос → короткий ответ |
| История обновлений | после каждого выпущенного пользователю изменения | новое сверху, с датой, 1–3 предложения о пользе |

Формулировки — для непрограммиста, польза вперёд, без дублей.

**Гранулярность истории:** каждое обновление — **отдельным блоком**, не склеивать. За день сделали 3 фичи → 3 отдельных блока (и цифра 3 на бейдже).

**Бейдж непрочитанных (в приложении):** на вкладке «Обновления» — красный кружок с **числом непрочитанных** обновлений (новых блоков с последнего открытия). Открыл вкладку → счётчик сбрасывается (кружок исчезает). Появилось новое обновление → кружок снова с числом. «Прочитано» хранится **на пользователя**. Контент истории при этом остаётся 1:1 сайт+приложение; на сайте бейдж опционален (localStorage).

**Правила:**
- Даты: все записи истории — с датой, сверху.
- Карточка маркетплейса / zip-инструкция — отдельный артефакт (часто лимит символов): синхронизировать вручную тем же контентом.
- Деплой: изменил источник → пересобрать и выложить **ОБЕ** поверхности.
- Не уверен, что/куда выносить — **спроси пользователя**, не решай молча.

## Чек-лист в конце каждой задачи

1. Затронуло пользователя? → обновить инструкцию/FAQ/историю в едином источнике.
2. История — с датой, сверху.
3. Есть карточка/zip — синхронизировать.
4. Выложить сайт И приложение.
5. Сомневаешься — спросить.
