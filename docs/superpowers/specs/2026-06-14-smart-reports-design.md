# Умные отчёты — дизайн / спецификация

- **Дата:** 2026-06-14
- **Статус:** согласовано на этапе brainstorming, готово к плану реализации
- **Продукт:** приложение для МоегоСклада, которое по расписанию формирует отчёты и отправляет их в Telegram (и на e-mail), а также показывает их внутри приложения.

---

## 1. Цель

Дать собственникам и руководителям бизнеса автоматические, осмысленные отчёты из МоегоСклада без необходимости заходить в МС и собирать цифры вручную. Приложение должно быть полезно максимально широкому кругу из 300 000+ пользователей МС — независимо от типа бизнеса.

Ключевая идея: **«поставил приложение → включил нужные базовые отчёты → подключил Telegram → получаешь».**

## 2. Пользователи

- **Основные:** собственники бизнеса, генеральные директора, руководители.
- **Дополнительные:** сотрудники — получают **личные** отчёты по своим показателям (свои продажи, свой заработок).
- Тип бизнеса — **любой**. Базовый набор универсален, специфика добавляется через конструктор.

## 3. Ключевые принципы

- **Отчёт — самостоятельная сущность.** У каждого отчёта свои: содержание, период, периодичность, форма доставки, получатели, состояние вкл/выкл.
- Несколько отчётов работают **параллельно**, каждый по своему расписанию.
- Отчёт может быть **общим** (одна сумма) или **с разбивкой** (по каналам, проектам, организациям, сотрудникам и т.д.).
- Отчёты **сгруппированы по группам** (см. каталог в приложении А).
- **Базовые отчёты = готовые пресеты конструктора**, предустановленные при установке приложения.
- Распространение и монетизация — **через маркетплейс МоегоСклада** (подписка). Детали монетизации отложены.

## 4. Архитектура

Выбран **гибридный подход** к данным: обычные отчёты считаются «живым» запросом к API МС в момент формирования; параллельно ведётся лёгкое хранилище ежедневных агрегатов для сравнений, трендов, сигналов и быстрого просмотра в приложении.

### Компоненты
- **Веб-интерфейс** (iframe внутри МС) — создание/настройка отчётов, управление получателями, лента и архив сформированных отчётов.
- **Хранилище конфигурации** — отчёты, расписания, получатели, правила заработка, пороги сигналов.
- **Планировщик** — определяет, когда какой отчёт пора формировать (часовой пояс аккаунта, «тихие часы»).
- **Воркеры отчётов** — по триггеру тянут данные из API МС, считают, форматируют.
- **Хранилище снимков** — ежедневные агрегаты по аккаунту: для сравнения со средним, трендов, быстрого просмотра.
- **Движок сигналов** — отслеживает события/пороги (вебхуки МС + периодический опрос) → мгновенные алерты.
- **Рендереры** — текст-карточка (с эмодзи), PDF, Excel, PNG-графики.
- **Доставка** — единый Telegram-бот (привязка по коду + отправка сообщений и файлов) и e-mail.

### Потоки данных
- **Обычный отчёт по расписанию:** расписание сработало → воркер берёт конфиг → запрос к МС API (+ снимки для сравнения) → расчёт → рендер форм → доставка получателям → сохранение в архив.
- **Сигнал:** событие из МС (вебхук/опрос) → проверка порога → если сработало → мгновенная отправка.
- **Снимки:** раз в день фоном собираются ключевые агрегаты по аккаунту в БД.

### Доступ к данным
- Приложение в маркетплейсе МС; при установке получаем доступ к аккаунту (токен), храним в зашифрованном виде.
- Данные берём из JSON API 1.2: эндпоинты `report/sales`, `report/profit`, `report/stock`, `report/turnover`, `report/money`, `report/dashboard`, `report/counterparty`, `report/orders` и документов (`demand`, `retaildemand`, `customerorder`, `supply`, `cashin`, `cashout`, `paymentin`, `paymentout`, `salesreturn` и т.д.).

## 5. Анатомия отчёта (конструктор)

Каждый отчёт — заполненная карточка по единой схеме. Базовые пресеты — предзаполненные карточки, которые можно менять.

1. **Идентификация:** название, группа, вкл/выкл.
2. **Метрики** (можно несколько): выручка/отгрузки, розничные продажи, заказы, прибыль и маржа %, остатки, обороты, движение денег, расходы по статьям, дебиторка/кредиторка, возвраты, средний чек, кол-во чеков и др.
3. **Группировка** (или «только общая сумма»): по каналу / проекту / организации / сотруднику / контрагенту / складу / точке / товару / группе товаров / типу товара / статье расходов / времени. Возможна вложенность (например, канал → товар).
4. **Фильтры «включить только» / «все кроме»** по каждому измерению: каналы, проекты, организации, сотрудники (в т.ч. «где ответственный = …»), контрагенты, склады/точки, группы и типы товаров, статусы документов.
5. **Период:** день / неделя / месяц / год / произвольный диапазон / скользящий («последние N дней»).
6. **База сравнения:** нет / прошлый аналогичный период / среднее за период / план (заданное число).
7. **Заработок сотрудника (опционально):** правило «% от [метрики] в рамках [каналов / где ответственный / контрагентов]» → личная сумма заработка.
8. **Форма:** текст / PDF / Excel / всё вместе.
9. **Расписание:** периодичность (день/неделя/месяц/год) + время + день недели/месяца. Один отчёт = одно расписание; для нескольких расписаний отчёт дублируется (в один клик).
10. **Получатели:** список (Telegram и/или e-mail). Тип доставки: **общий** (всем одинаковый) или **личный** (каждому свой срез).

Пример отчёта, который покрывается этой схемой: *«Отгрузки за май → каналы 1, 3, 5 → где ответственный Иванов → все контрагенты КРОМЕ "Меркури Мода ООО" → все товары КРОМЕ услуг».*

## 6. Умные функции (что делает приложение незаменимым)

Все включены в ядро.

- **A. Сравнение (настраиваемая база).** Рядом с каждой цифрой ±% и стрелка 🟢🔴. База выбирается: прошлый аналогичный период (для ежедневного — вчера, для недельного — прошлая неделя и т.д.), среднее за период, заданный план, или без сравнения.
- **B. Сигналы/алерты (по событиям и порогам, работают постоянно).** Крупная продажа, продажа ниже себестоимости, товар закончился/ниже минимума, резкое падение выручки, крупный возврат, просрочка платежа. Пороги настраиваются.
- **C. План/факт.** Цель (выручка, маржа и т.п.) → отчёт показывает % выполнения.
- **D. AI-комментарий.** Короткий вывод человеческим языком («Выручка ниже среднего на 15%, главная причина — канал Ozon −40%»).
- **E. Топы и антирейтинги.** Автоматически «лучший/худший» товар/клиент/менеджер в отчёте.
- **Заработок сотрудника.** См. п.5.7 — личная сумма заработка по правилу %.

## 7. Базовый набор (предустановлено при установке)

11 готовых пресетов, которые нужно только включить и выбрать получателей.

**Сводные (флагманы):**
1. **«Доброе утро» — сводка за вчера.** Выручка, прибыль, маржа %, чеки, средний чек, приход/расход денег, остаток на счетах, топ-товар, главные сигналы. *Ежедневно 9:00.*
2. **Итоги дня.** То же за сегодня. *Ежедневно 20:00.*
3. **Недельный / месячный дайджест KPI.** Ключевые цифры со сравнением к прошлому периоду. *Пн 9:00 / 1-е число.*

**Продажи и прибыль:**
4. **Отгрузки/продажи за период** — общая сумма + разбивка по каналам / проектам / организациям. *Ежедневно.*
5. **Прибыль и маржа** — прибыль, рентабельность %, разбивка по каналам + топ и антирейтинг товаров по марже. *Ежедневно/еженедельно.*

**Деньги:**
6. **Деньги: приход/расход и остатки** — движение денег, остатки по счетам и кассам, расходы по статьям. *Еженедельно.*
7. **Дебиторская задолженность** — кто должен, топ должников, просрочка. *Пн утром.*

**Склад:**
8. **Остатки и «что заказать»** — стоимость склада + товары ниже минимума / закончившиеся. *Ежедневно/еженедельно.*

**Люди и клиенты:**
9. **Отчёт по сотрудникам** — продажи/прибыль по каждому, рейтинг (+ опция личного отчёта сотруднику с его заработком). *Еженедельно/ежемесячно.*
10. **Топ клиентов** — лидеры по выручке/прибыли + новые клиенты. *Ежемесячно.*

**Сигналы:**
11. **Алерты** — крупная продажа, продажа ниже себестоимости, товар закончился, резкое падение выручки, крупный возврат, просрочка платежа. *В реальном времени, пороги настраиваются.*

## 8. Доставка и получатели

- **Единый бот** (рабочее название `@UmniyOtchetBot`). Получатель жмёт «Старт», вводит код привязки из приложения → его Telegram привязывается к профилю получателя. Один раз.
- **Профиль получателя:** имя + канал доставки (Telegram и/или e-mail); для личных отчётов — связь со сотрудником МС.
- **Личные отчёты сотрудникам:** настраивается один раз, приложение для каждого получателя-сотрудника считает его срез (его продажи / где он ответственный / его заработок) и шлёт только ему.
- **Формы:** в сообщении — текст с эмодзи (карточка с цифрами и сравнением), вложением — PDF и/или Excel. E-mail — то же содержимое с вложениями. В приложении — лента и архив: открыть онлайн, скачать, переслать; история хранится.
- **Удобства:** кнопка «Отправить сейчас» (тест до постановки на расписание); часовой пояс аккаунта; «тихие часы».

## 9. Расписание

- Периодичность: день / неделя / месяц / год.
- Время + день недели/месяца.
- Учитывается часовой пояс аккаунта и «тихие часы».
- Сигналы работают вне расписания (реальное время).

## 10. Обработка ошибок и пограничные случаи

- **Лимиты API МС** → очередь + ретраи с бэкоффом; тяжёлые запросы разносятся во времени.
- **Сбой доставки** (бот заблокирован, неверный e-mail) → ретрай + пометка в приложении.
- **Нет данных за период** → корректный отчёт «нет данных», без падения.
- **Токен отозван / подписка кончилась** → отчёты на паузу + уведомление в приложении.

## 11. Приватность и безопасность

- Храним минимум: агрегаты-снимки, не сырые документы.
- Токены шифруются.
- Доступ строго в рамках аккаунта МС.

## 12. Масштаб

- Многотенантность под 300 000+ потенциальных аккаунтов.
- Расписания распределяются во времени, обработка через очереди.

## 13. Открытые вопросы (отложено)

- **Монетизация:** подписка через маркетплейс МС; конкретные тарифы и разбивка «что бесплатно / что платно» — позже.
- Модель доступа к API при установке (детали Vendor API) — уточнить на этапе плана.
- Рабочее имя бота и брендинг.

## 14. Вне рамок на старте (YAGNI)

- Свой бот у каждой компании (white-label) — возможно позже как опция.
- Полный каталог ~100 отчётов как предустановленные — нет; они доступны через конструктор, предустановлены только 11.

## Приложение А. Полный каталог отчётов (reference, ~100, по группам)

Доступны через конструктор; базовые 11 — предустановлены.

**A. Продажи и выручка:** продажи за период; по каналам; по проектам; по контрагентам; по розничным точкам; по организациям; по складам; по сотрудникам; динамика по дням/неделям/месяцам; по часам/дням недели; сравнение периодов; план/факт; возвраты покупателей; розница по сменам; структура оплат.

**B. Прибыль и маржинальность:** общая прибыльность; по товарам; по модификациям; по контрагентам; по сотрудникам; по каналам; по группам/категориям; топ маржинальных/убыточных; по проектам/точкам.

**C. Товарная аналитика:** топ по количеству; топ по выручке; ABC; XYZ; по группам; по категориям/подкатегориям; по брендам; неликвиды; оборачиваемость; товары без движения.

**D. Остатки и движение:** текущие остатки; по складам; ниже минимума; нулевые/отрицательные; резерв/ожидание; доступно к продаже; обороты; списания; оприходования; перемещения; инвентаризации; стоимость запасов в динамике.

**E. Закупки и поставщики:** закупки за период; по поставщикам; заказы поставщикам; возвраты поставщикам; товары к заказу; динамика закупочных цен.

**F. Деньги (cash flow):** движение денег; остатки по счетам/кассам; приход по статьям; расход по статьям; поток по проектам/организациям; платёжный календарь; кассовая книга по точкам.

**G. Расходы:** по статьям; по категориям; по проектам; по контрагентам; по организациям; выплаты сотрудникам; динамика и структура расходов; упрощённый P&L.

**H. Взаиморасчёты:** дебиторка; кредиторка; просроченная дебиторка; по договорам; топ должников.

**I. Клиенты:** новые клиенты; топ клиентов; реактивация/отток; RFM; средний чек по сегменту; показатели контрагента.

**J. Сотрудники:** рейтинг менеджеров; выполнение плана; конверсия заказов; выданные скидки.

**K. Точки/каналы/проекты:** сравнение точек; эффективность канала; сводка по проекту (мини-P&L).

**L. Заказы и воронка:** заказы за период; воронка по статусам; зависшие заказы; к отгрузке сегодня/завтра; неоплаченные счета; статистика заказов.

**M. Сводные дашборды:** утренняя сводка; вечерняя сводка; недельный/месячный дайджест; показатели дашборда МС; сводный KPI-борд.

**N. Сигналы/алерты:** крупная продажа; товар ниже минимума; продажа ниже себестоимости; резкое падение/рост выручки; крупный возврат; новый крупный клиент; просрочен платёж.
